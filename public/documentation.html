APP TITLE 'MyFLIX'

* The web application will provide users with access to information about different movies, directors, and genres.

* Users will be able to sign up, update their personal information, and create a list of their favorite movies.

REQUEST URL Http Method Response
Return a list of ALL movies to the user /movies GET returns json objects of all movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/:Title GET returns json object of selected movie title
Return data about a genre (description) by name/title (e.g., “Thriller”) /genre/:Name GET returns json object of selected genre
Return data about a director (bio, birth year, death year) by name /directors/:Name GET returns json array containing bio/dob/dod keys and values
Allow new users to register /users POST adds a new user json object created in the request body to users register.
exp. format:
{
"Username" : "Big Foot", "Password" : "Himalaya", "Email" : "Foooooooooot@gmail.com", "FavouriteMovies" : "", "Birthday" : "1974-12-05"
}
Allow users to update their user info /users/:Username PUT updates user data. Update user json object in request body.
exp. format:
{
"Username": "Mr T", "Password": "No Fly", "Email": "TheATeam@gmail.com", "Birthday": "1984-10-20"
}
Allow users to add a movie by MovieID to their list of favorites. /users/:Username/movies/:MovieID POST adds selected movie title to favourites list.
Shows updated user json object, including FavouriteMovies list by id.
Movies already on the list will not be added due to usage of $addToSet
Allow users to remove a movie by MovieID from their list of favorites /users/:Username/movies/:MovieID DELETE removes selected movie from favourites list
Shows updated user json object, including FavouriteMovies list by id.
Allow existing users to deregister. /users/:Username DELETE deletes user from register by Username
If successful shows "Username was deleted" otherwise "Username was not found"
